> ARCH-07 讨论确认 | 2026-07-18

## 概述

摸鱼面板内嵌的经典 2048 数字消除游戏。轻量消遣玩法，通过分数分段递减 + 首达阶梯奖励获取经验和金币，与宠物心情体系联动。

## 定位

| 维度 | 说明 |
|------|------|
| 角色 | 面板内轻量消遣，经验/金币的补充获取渠道 |
| 窗口 | 面板内嵌，不独立开窗 |
| 收益控制 | 分数分段递减 + 首达阶梯，无每日上限 |
| 宠物联动 | 心情加成，属性自然衰减不暂停 |

---

## 一、游戏核心规则

### 1.1 标准 2048 玩法

- 4×4 网格，初始随机生成 2 个方块（2 或 4）
- 每次滑动（上下左右），所有方块向该方向移动，相邻相同数字合并，合并后方块值为两者之和
- 合并后随机在空格生成新方块（90% 概率 2，10% 概率 4）
- 无合法移动且无空格 → Game Over
- 计分方式：每次合并，累加合并后方块的值（如 4+4 → 8，加 8 分）

### 1.2 操作方式

- 键盘方向键（↑↓←→）
- 鼠标拖拽（在游戏区域按住拖拽到四个方向，松手触发滑动）

### 1.3 到达 2048 后

不强制结束，可以继续玩冲更高分（4096、8192...）。

### 1.4 重新开始

任意时刻可点击「重新开始」按钮。点击后弹出确认弹窗："当前对局进度将丢失，确定重新开始？"。确认后重置棋盘，本局不计任何收益。

## 二、收益结算规则

Game Over 后自动结算，弹出结算弹窗。

### 2.1 基础收益：分数分段递减

分数越高，单位分数的收益越高（与游戏后期难度递增匹配）：

| 分数段 | 经验兑换 | 金币兑换 |
|--------|----------|----------|
| 0 ~ 1000 | 每 200 分 = 1 exp | 每 300 分 = 1 coin |
| 1001 ~ 3000 | 超出 1000 部分，每 120 分 = 1 exp | 超出 1000 部分，每 180 分 = 1 coin |
| 3000+ | 超出 3000 部分，每 80 分 = 1 exp | 超出 3000 部分，每 120 分 = 1 coin |

**余数处理**：各段计算结果四舍五入到整数。

**所有分段阈值与兑换比例均配置化**。

### 2.2 首达阶梯奖励（一次性）

首次合成指定数字的方块，发放一次性奖励。每个档位终身仅生效一次：

| 方块 | 经验 | 金币 |
|------|------|------|
| 128 | +5 | +3 |
| 256 | +10 | +6 |
| 512 | +20 | +12 |
| 1024 | +40 | +25 |
| 2048 | +80 | +50 |

达成标记存入本地存档，已领过的档位不再重复触发。

### 2.3 结算流程

1. Game Over → 自动弹出结算弹窗
2. 弹窗展示：本局得分、历史最高分、本局经验/金币收益明细、首达奖励解锁情况（如有）
3. 用户点击「再来一局」或「返回」→ 收益实时到账
4. 全局经验、金币、面板数值同步刷新

**注意**：「再来一局」和「返回」都会发放收益。用户只是确认查看明细。

## 三、宠物养成联动

### 3.1 心情收益加成

结算时根据宠物当前心情档位，对本局总收益（基础收益 + 首达奖励）施加倍率：

| 心情档位 | 心情值范围 | 倍率 |
|----------|-----------|------|
| 开心 | ≥ 80 | +20%（即 ×1.2） |
| 良好 | 50 ~ 79 | 正常（×1.0） |
| 一般 | 30 ~ 49 | 正常（×1.0） |
| 低落 | < 30 | -30%（即 ×0.7） |

**加成系数配置化**。

### 3.2 属性自然衰减

游戏过程中，宠物的饱腹值、心情值自然衰减正常生效（与打开普通面板逻辑一致），不做特殊暂停。

---

## 四、游戏状态持久化

| 场景 | 行为 |
|------|------|
| 面板内导航切页（主页 ↔ 仓库 ↔ 商店 ↔ 设置 ↔ 2048） | **恢复**棋盘（同渲染进程，内存保留） |
| 关面板回到宠物态，再双击打开面板 | **恢复**棋盘（通过 PetState 持久化） |
| 退出 App 重启 | **重置**棋盘（主进程启动时清除 2048 存档） |

**恢复内容**：棋盘状态（4×4 格子值）、当前分数、Game Over 标记。

**恢复时如果上局已 Game Over**：直接展示结算弹窗（收益已在上次结算时发放，不会重复）。

> ⚠️ 实现注意：App 重启时主进程需清除 game2048.savedGame，否则关面板→重启→开面板会错误恢复旧局。

## 五、页面与交互

### 5.1 入口

导航栏新增「🎮 2048」菜单项，位于商店下方。首期直接进 2048（不做二级"小游戏"菜单，等第二个游戏再加）。

### 5.2 游戏界面（由上到下）

1. **顶部信息栏**：
   - 左侧：本局实时分数
   - 中间：历史最高分
   - 右侧：当前心情收益倍率（如 `心情加成 +20%`）

2. **中部游戏区**：
   - 经典 4×4 网格
   - 方块样式适配面板暗色主题
   - 不同数值用差异化颜色区分
   - 支持键盘方向键 + 鼠标拖拽

3. **底部操作区**：
   - 「重新开始」按钮（任意时刻可点，需确认弹窗）

### 5.3 结算弹窗

Game Over 后自动弹出，内容包括：
- 本局得分
- 历史最高分（如破纪录可标注 🏆）
- 经验收益明细（基础 + 首达 + 心情加成后合计）
- 金币收益明细（基础 + 首达 + 心情加成后合计）
- 新解锁的里程碑（如有，如 `🎉 首次合成 256！`）

底部按钮：
- 「再来一局」→ 收益到账，清空棋盘开始新局
- 「返回」→ 收益到账，回到面板主页

## 六、数据结构

```js
// store.js DEFAULT_STATE 新增
game2048: {
  highScore: 0,           // 历史最高分
  milestones: {           // 首达阶梯标记（达成后置 true）
    128: false,
    256: false,
    512: false,
    1024: false,
    2048: false,
  },
  savedGame: null,        // 关面板时保存的棋盘（App 启动时清除）
                          // { board: number[][], score: number, gameOver: boolean }
}
```

- `savedGame` 仅在面板关闭时写入（PetState），App 启动时主进程清除
- 面板内导航切页不读写 PetState（同一渲染进程，内存保留即可）
- 首达标记跨会话持久化，不会丢失
